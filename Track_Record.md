# Track Record — 逻辑验证记录

&gt; **情报的价值在于领先于共识的"对账成功"。**
&gt; 
&gt; 本文件按四层闭环架构维护：输入层（信源分级）→ 处理层（三轨并行）→ 输出层（论证强度）→ 反馈层（证伪与回溯）。
&gt; 所有记录均可基于本仓库开源证据链复现与审计。

---

## 记录规范

| 字段 | 说明 |
|------|------|
| **日期 (UTC+8)** | 信号捕捉时间 |
| **异常信号** | OSINT 捕捉到的偏离共识的异常 |
| **逻辑判定** | 基于四层架构推导的结论 |
| **论证强度** | `确认性判断` / `方向性判断` / `推测性观察` |
| **信源路径** | 可复现的证据链入口（URL / 文档编号 / 模型输出标识） |
| **验证状态** | `✅ 已验证` / `⏳ 监测中` / `⏳ 信源待核` / `❌ 证伪` |
| **不可知维度** | 该结论依赖但超出公开信息验证能力的变量 |
| **证伪条件** | 何种公开信号出现即可推翻该判定 |

## 历史对账存证

本仓库 `/Proof_of_Timestamp/` 目录保留预判时间戳与事实链的交叉验证记录。每条记录包含：
- 预判原文截图或文本存档
- 事实发生后的公开报道链接
- 偏差分析（若预判与事实不符）

&gt; **原则**：验证失败不删除记录，而是标注"预判偏差原因"，纳入下一轮 SEJ 校准。
